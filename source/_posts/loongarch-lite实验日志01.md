# loongarch-lite实验日志01
目标:完善TEMU的基本功能
## TEMU的阅读和修改
### TEMU简介
TEMU是一个轻量级的动态二进制仿真器,能够仿真对应平台的二进制机器码(应该是在原天大小学期程设实践课程的部分,因故移动至此).
其基本功能和调试器类似,需要实现类似单步执行,设置断点之类的功能.

### 代码框架
TEMU针对计算机中不同的硬件部分分别实现了接口,其基本结构如下:
```
temu
├─ include
│  ├─ common.h  # 全局使用
│  ├─ misc.h    # 杂项
│  ├─ debug.h   # debug使用
│  ├─ macro.h   # 便捷宏定义
│  ├─ temu.h    # 核心定义
│  ├─ memory    # 内存头文件
│  ├─ monitor   # 调试命令头文件
│  └─ cpu       # 处理器头文件
└─ src
   ├─ cpu
   │  ├─ 3r-type.c
   │  ├─ exec.c         # MIPS32
   │  ├─ i12-type.c
   │  ├─ i20-type.c
   │  ├─ reg.c          # 寄存器实现
   │  └─ special.c      # 特殊指令
   ├─ main.c            # 函数入口(含调用初始化)
   ├─ memory            # 内存相关实现
   │  ├─ dram.c
   │  └─ memory.c
   └─ monitor           # 调试器实现
      ├─ cpu-exec.c     # 指令执行的主循环
      ├─ expr.c         # 表达式求值实现
      ├─ monitor.c      
      ├─ ui.c           # 交互命令行实现
      └─ watchpoint.c   # 监视点实现

```  
实现初期,首先阅读`main.c`函数代码,发现三个封装部分,分别是
1. 调试器本体的初始化
2. 调试器模拟启动计算机
3. 进入调试器UI界面

```
int main(int argc, char *argv[]) {
	//带参数的监视器初始化
	init_monitor(argc, argv);

	//模拟启动计算机系统
	restart();

	//循环取指的UI界面
	ui_mainloop();

	return 0;
}
```
### 单步实现

由于模拟启动计算机的过程较为复杂(指导书语,事实上也是),可以先从调试器本体或者UI主循环开始考虑,这里我们先从调试器开始

阅读ui.c,可以照猫画虎,按照已经实现的命令增加help中的命令列表,并定义相关函数.
然后参考已经实现的`c`命令,发现其的实现函数及其简单,仅仅是向`CPU执行函数`传入了-1的参数
```
static int cmd_c(char *args) {
	cpu_exec(-1);
	return 0;
}
```
接下来顺其阅读`cpu_exec`函数,发现其接受u32的n参数,并且参与循环的判断.如果传入-1的参数,无符号类型会溢出成为极大值,间接实现了`执行到程序结束`这项功能.不得不说,这个部分的处理十分优雅,如果是我来写的话,只会传入0这种参数,然后用if做逻辑控制判断,手动设置最大的执行次数,这里也能看出往昔程序员对计算机系统的熟悉和优雅的编码风格.

于是实现指定步数的单步调试就很容易了,给`cpu_exec`函数传入指定步数即可.

### 表达式实现
本来这里我初步实现了内存扫描,信息打印等功能,但是考虑到他们的完整实现需要表达式的实现,就先介绍表达式计算的实现.
对于一个计算表达式
```
2*(3+4)-5
```
我们应该如何去计算它呢?可以按如下方法先拆分这个式子里的各个元素
```
2  参与计算的数字
*  运算符
(  运算符
3  参与计算的数字
+  运算符
4  参与计算的数字
)  运算符
-  运算符
5  参与计算的数字
```
我们所拆出的每个元素就可以称之为一个 `token` ,每个token有不同的类型,对不同类型的token进行不同规则的运算,这就是词法分析的基本原理.

TEMU中使用正则表达式来完成对token的匹配,将一段字符串解析成各个token,然后按照对应的规则进行计算,计算部分将在下个日志完成.
